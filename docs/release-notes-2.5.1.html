🔍 背景：为什么需要这个版本

v2.1.0 将完成检测从 marker 标记协议（AI 主动输出 ⚡ARENA_START⚡/ARENA_DONE⚡）切换到 文本稳定 + streaming UI 检测，方向正确但落地不完整。

核心风险 静默失败：把用户刚发的 prompt 当成 AI 回复读回 StateMachine → 下一轮辩论拿到垃圾上下文，无报错，用户难以发现。

Fix 2 · 防护层
hasStreamedMap — 必须"流过"才准判 ready

旧逻辑：textLength > 0 && !lengthChanged && !isStreaming — 第一次轮询就可能满足（isStreaming 选择器失效 → 永远 false + 文本长度=用户消息字数不变）→ 1.5s 误判 ready。

BEFORE — sidepanel.js:403
if (v.textLength > 0 && !lengthChanged && !v.isStreaming) {
  stableCounts[id] = (stableCounts[id] || 0) + 1;
  // ... 3 次稳定 → ready
}
AFTER — 状态机：IDLE→流式→稳定→就绪
// 观察流式信号
if (lengthChanged || v.isStreaming) hasStreamedMap[id] = true;
// ready 前置条件：必须流过
if (textLength > 0 && !lengthChanged && !isStreaming
    && hasStreamedMap[id]) {
  stableCounts[id] = (stableCounts[id] || 0) + 1;
}

Fix 3 · 最后防线
Sanity Check — 提取后三段校验

旧逻辑：readOneResponse 拿到文本直接 setParticipantResponse，零校验。

BEFORE — background.js:419-428
const text = r?.text || "";
if (text) {
  StateMachine.setParticipantResponse(p.id, text);
}
return { ok: true, text };
AFTER — 三层拒绝 + 状态回退
// 1. 与 prompt 完全相同 → 拒绝
if (sent && text === sent)
  return { ok: false, error: "疑似读到用户消息..." };
// 2. 前 100 字与 prompt 相同 → 拒绝
if (sent && head(text) === head(sent))
  return { ok: false, error: "疑似读到用户消息..." };
// 3. 与上轮 response 相同 → 拒绝
if (prevResp && text === prevResp)
  return { ok: false, error: "疑似读到残留回复..." };
// ✓ 通过 → 写入
StateMachine.setParticipantResponse(p.id, text);

+ 自动提取被拒绝时 sidepanel 回退 _pollStatus→waiting + 清 stableCounts + 重置 hasStreamedMap，避免死锁。

👤 用户体验变化

🟢

ChatGPT 不再"假就绪"

修复前：ChatGPT 发送后 1.5 秒就可能直接跳 ✅（跳过 streaming 阶段）。 修复后：必须经过 ⏳ streaming → 长度稳定 → ✅ ready 完整流程。

🛡️

不会把用户问题当成 AI 回答

修复前：选择器误匹配或 heuristic 兜底时，静默把用户 prompt 读成 AI 回复。 修复后：sanity check 三段校验拦截，sidepanel 显示 "疑似读到用户消息，请手动提取"。

🔄

提取失败自动恢复

修复前：ready 判定后如果提取失败，状态卡死。 修复后：自动回退 waiting + 清稳定计数 + 重置流式标记，轮询继续观察。

📊

文本长度更准确

textContent → innerText，9 平台统一使用"用户可见文本"做流式长度追踪，不再把隐藏 DOM 的内容算进去。

🎯

Kimi / Yuanbao 选择器更准

裸 [class*="markdown"] 可能命中用户消息 → [class*="assistant"] [class*="markdown"] 只匹配 AI 回复区。

⏱️

无感知变化

版本号 2.5.0 → 2.5.1，不影响现有工作流。图片注入、PPT 模式、导出、hardReset 均回归通过。
